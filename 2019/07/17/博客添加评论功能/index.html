fluid主题：valine评论

修改配置文件

修改fluid主题目录下的_config.yml文件，如下所示开启评论并选择使用的评论模块为valine

comments:  # 评论
    enable: true  # 开启评论
    type: valine

然后修改valine模块中中的配置信息, 主要是修改appid、appkey、serverURLS这三处信息, 需要 注册LeanCloud才能获取appid、appkey, 具体操作可参考文档

# Valine
# 完整文档 https://valine.js.org/configuration.html
# 注意：下列配置项中的 true/false 不要用引号括起来
valine:
  appid: xxx  # 从 LeanCloud 的应用中得到的 appId
  appkey: xxx  # 从 LeanCloud 的应用中得到的 APP Key
  placeholder: 说点什么 # 评论框占位提示符
  path: window.location.pathname # 当前文章页路径，用于区分不同的文章页，以保证正确读取该文章页下的评论列表
  avatar: retro # Gravatar 头像展示方式
  meta: ['nick', 'mail', 'link']  # 评论者相关属性
  pageSize: 10 # 评论列表分页，每页条数
  lang: zh-CN # zh-CN | zh-TW | en | ja
  highlight: false # 代码高亮
  recordIP: false # 是否记录评论者IP
  serverURLs:  xxx # REST API 服务器地址，国际版不填

评论邮件提醒功能

邮件提醒我使用的是Valine-Admin，参考文档进行Git部署即可。但我在部署时出现了如下错误：

主要是因为Node版本太低了的缘故，需要修改项目中package.json文件中的Node版本，可以将项目fork到自己的仓库中，然后进行修改, 将6.x修改成12.x即可。

"engines": {
    "node": "12.x"
  }

注意

如果fluid主题使用了覆盖配置，一定记得修改配置时是在source/_data目录下修改，不然配置会无效。
